## Overview

The Synchronizer keeps your browser profiles in sync across multiple devices. When you modify a profile on one machine — updating cookies, changing settings, or adding bookmarks — those changes are automatically synced to the cloud and available on your other devices.

## How it works

1. **Profile data** (cookies, localStorage, fingerprint) is encrypted and uploaded to the Oxygen Atmos cloud after each session
2. When you open a profile on another device, the **latest data is downloaded** automatically
3. **Conflict resolution** uses last-write-wins — the most recent session takes priority

## Sync settings

Navigate to **Settings** → **Synchronizer** to configure:

| Setting | Description | Default |
|---|---|---|
| **Auto-sync** | Sync profiles automatically after closing | Enabled |
| **Sync on launch** | Download latest data when opening a profile | Enabled |
| **Sync frequency** | How often to sync running profiles | 5 minutes |
| **Selective sync** | Choose which profiles to sync | All profiles |

## Manual sync

To force a sync manually:

1. Right-click a profile in the list
2. Select **Sync Now**
3. Wait for the sync indicator to confirm completion

<Info>
  Sync data is encrypted end-to-end. Oxygen Atmos servers cannot read your profile data.
</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Profile data is outdated on another device">
    Check that auto-sync is enabled on both devices. Try a manual sync on the source device, then re-open the profile on the target device.
  </Accordion>
  <Accordion title="Sync is slow">
    Large profiles with extensive cookies and cache take longer to sync. Consider clearing unnecessary browsing data before syncing.
  </Accordion>
  <Accordion title="Sync conflicts">
    If two devices modify the same profile simultaneously, the last session to close wins. Avoid opening the same profile on multiple devices at the same time.
  </Accordion>
</AccordionGroup>
