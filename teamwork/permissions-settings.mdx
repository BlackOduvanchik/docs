## Overview

Permissions control what team members can do with shared profiles and folders. The team owner and admins can set permissions per member or per folder.

## Permission levels

| Permission | Description |
|---|---|
| **View** | See profile details and fingerprint settings |
| **Edit** | Modify profile settings, proxy, and notes |
| **Launch** | Start and stop the browser profile |
| **Transfer** | Send profiles to users outside the team |
| **Delete** | Remove profiles from the shared folder |

## Setting permissions

<Steps>
  <Step title="Open shared folder settings">
    Navigate to the shared folder → click the **Permissions** icon.

    {/* Screenshot: folder permissions icon */}
  </Step>
  <Step title="Select a member">
    Choose the team member to configure.
  </Step>
  <Step title="Toggle permissions">
    Enable or disable individual permissions for the selected member.

    {/* Screenshot: permission toggles */}
  </Step>
  <Step title="Save">
    Click **Save** to apply the permissions.
  </Step>
</Steps>

## Default permissions

When a new member joins a team, they receive default permissions:

- **View** — enabled
- **Edit** — disabled
- **Launch** — enabled
- **Transfer** — disabled
- **Delete** — disabled

Admins can change default permissions in **Teamwork** → **Settings** → **Default Permissions**.

<Info>
  Admins always have full permissions on all shared folders. Permission settings only apply to members with the **Member** role.
</Info>

## Permission inheritance

Permissions are set at the **folder level**. All profiles inside a folder inherit the folder's permission settings. To give different access to different profiles, organize them into separate folders.
