## Overview

Oxygen Atmos teamwork features let multiple users collaborate on the same set of browser profiles. Create a team, invite members, assign roles, and control who can access, edit, or launch specific profiles.

## Key concepts

| Concept | Description |
|---|---|
| **Team** | A group of Oxygen Atmos users working together |
| **Owner** | The user who created the team — full control |
| **Admin** | Can manage members and most settings |
| **Member** | Can access shared profiles based on permissions |
| **Shared folder** | A folder visible to team members with assigned access |

## What you can share

- **Browser profiles** — share individual profiles or entire folders
- **Proxies** — make proxies available to team members
- **Extensions** — share global extensions with the team

{/* Screenshot: team dashboard overview */}

## How sharing works

1. The team owner or admin **shares a folder** with the team
2. Members with access can **view, edit, or launch** profiles in that folder (based on their permissions)
3. All changes are **synced in real-time** — if a member modifies a profile, others see the update

<Info>
  Team members need their own Oxygen Atmos subscription. Shared profiles don't count against member quotas — only the owner's.
</Info>
